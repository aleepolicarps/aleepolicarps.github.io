Sessions

A way to store information (in variables) to be used across multiple pages where the information is not stored on the users computer.

What is a session?

  • You open the application
  • Do some operations
  • Save changes
  • Close application

Starting the session session_start

We use the superglobal $_SESSION to store the sessions variables.

Note: The session_start() function must be the very first thing in your document. Before any HTML tags.

Example.
Create a php file named session1.php and copy the following contents.

<?php  
// Start the session  
session_start();  
?>  
<!DOCTYPE html>  
<html>  
<body>  
  
<?php  
// Set session variables  
$_SESSION["background_color"] =  "green";  
$_SESSION["text"] =  "I amn from the SESSION";  
echo  "Session variables are set.";  

?>  
  
</body>  
</html>

Next, create another php file and name it session2.php and copy the following contents.

<?php  
session_start();  
?>  

<!DOCTYPE html>  
<html>  
<body bgcolor="<?php echo $_SESSION['background_color']; ?>">  
	 <h1><?php echo $_SESSION['text']; ?></h1>
</body>  
</html>

Notice that the variables are not passed directly via request to the next page but are retrieved from the $_SESSION superglobal.

Note
$_SESSION, while a superglobal, is still an array. You can get and print all the variables stored in the session by using the function print_r

<?php

print_r($_SESSION);

Modifying session variables

To modify a session variable, simple re-assign a value to the index.

$_SESSION['background_color']

Destroying a PHP Session

If you wish to terminate the session (e.g. logout the user), you can call the functions session_unset() and session_destroy().
